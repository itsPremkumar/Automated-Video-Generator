# Repository Growth Checklist

These are the highest-impact growth actions that still need to be done manually in GitHub or during launch.

## GitHub settings

- Set the repo description to a clear search-friendly sentence
- Add the website URL if you deploy the portal publicly
- Add relevant repository topics
- Upload a social preview image
- Enable Discussions if you want product and workflow questions off the issue tracker
- Pin 1 to 3 important issues, such as roadmap, showcase, or good first issue

## Content and trust

- Record and upload a short demo video or GIF to the README
- Create a first tagged release so the project feels maintained
- Add real output examples from YouTube Shorts, TikTok, or explainer workflows
- Keep the README headline and first paragraph focused on who the project is for
- Keep CI green so contributors trust the repo

## Community loops

- Label beginner-friendly issues as `good first issue`
- Label roadmap items as `help wanted`
- Reply quickly to the first few contributor issues and PRs
- Thank contributors publicly in release notes or README sections

## Distribution

- Share the repo on X, LinkedIn, Reddit, Dev.to, Hashnode, and relevant Discord communities
- Post a before/after demo clip when sharing
- Publish a short tutorial on how to generate a YouTube Short in minutes
- Submit the project to open-source directories and Remotion or MCP ecosystem lists

## Assets included in this repo

- `assets/github-social-preview.svg` can be exported to PNG and uploaded as the GitHub social preview image
- `README.md` already contains improved star and search messaging
- `.github/workflows/ci.yml` adds a visible CI badge opportunity
